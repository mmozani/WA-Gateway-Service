# 🚀 WA-Gateway-Service (WhatsApp Multi-Session API)

یک میکروسرویس حرفه‌ای، ماژولار و امن مبتنی بر **Node.js** که سرور یا اپلیکیشن‌های شما (Laravel، Python و ...) را مستقیماً به هسته واتس‌اپ متصل می‌کند. 

این سیستم با مدیریت همزمان چندین نشست (Session) و دور زدن محدودیت‌های مسدودسازی (Anti-Ban)، یک گیت‌وی پایدار برای ارسال پیام‌های سیستمی و OTP خلق می‌کند.

---

## 🌍 معجزه این سرویس برای کسب‌وکارهای ایرانی چیست؟

بزرگترین چالش استارتاپ‌ها و وب‌سایت‌های ایرانی، **احراز هویت و ارتباط با کاربران بین‌المللی (خارج از ایران)** است. این سرویس دقیقاً برای حل این بحران طراحی شده است:

1. **کاهش ۱۰۰ درصدی هزینه‌های ارزی:** هزینه ارسال هر پیامک (SMS) بین‌المللی به شدت بالاست و با نوسانات ارز تغییر می‌کند. با این سرویس، ارسال پیامک OTP به یک کاربر در اروپا یا آمریکا **کاملاً رایگان** تمام می‌شود.
2. **دور زدن تحریم‌های سرویس‌دهنده‌ها:** سرویس‌هایی مثل *Twilio* یا *MessageBird* به کسب‌وکارهای ایرانی خدمات نمی‌دهند و حساب‌ها را مسدود می‌کنند. شما با این سیستم، گیت‌وی اختصاصی و تحریم‌ناپذیر خودتان را می‌سازید.
3. **تضمین دلیوری (Delivery):** پیامک‌های بین‌المللی اغلب به دلیل فایروالِ اپراتورهای خارجی به دست کاربر نمی‌رسند، اما پیام واتس‌اپ مستقیماً و در کسری از ثانیه به گوشی کاربر می‌رسد.
4. **رهایی از مسدودسازی واتس‌اپ (Anti-Ban):** ارسال ماشینی و سریع پیام باعث مسدود شدن خط می‌شود. ما با پیاده‌سازی مکانیزم **تأخیر تصادفی (Human-like Delay)** رفتار انسانی را شبیه‌سازی کرده‌ایم تا خطوط شما مسدود نشوند.

## 🛠 آموزش دقیق و گام‌به‌گام نصب (لینوکس / اوبونتو)

برای نصب خودکار این سرویس روی سرور لینوکس، نیازی به دانش فنی پیچیده ندارید. اسکریپت `setup.sh` تمام پیش‌نیازها، کتابخانه‌های گرافیکی کرومیوم، Node.js و PM2 را برای شما نصب می‌کند.

### مرحله اول: دریافت پروژه
ابتدا از طریق ترمینال به سرور خود متصل شوید و مخزن را کلون کنید:
```bash
git clone [https://github.com/mmozani/WA-Gateway-Service.git](https://github.com/mmozani/WA-Gateway-Service.git)
cd WA-Gateway-Service
```

### مرحله دوم: نصب خودکار با یک کلیک (One-Click Setup)
با اجرای این دستور، تمام نیازمندی‌های لینوکس نصب شده و فایل تنظیمات `.env` برای شما ساخته می‌شود:
```bash
chmod +x setup.sh && ./setup.sh
```

### مرحله سوم: تنظیمات امنیتی
حالا فایل `.env` را باز کنید و کلید امنیتی خود را تنظیم کنید:
```bash
nano .env
```
*(در این فایل مقدار `SECRET_API_KEY` را به یک رمز قوی تغییر دهید و با `Ctrl+X` و سپس `Y` آن را ذخیره کنید).*

### مرحله چهارم: اجرای دائم سرویس
برای اینکه سرویس در بک‌گراند اجرا شود و حتی با ریستارت شدن سرور هم خاموش نشود:
```bash
pm2 start ecosystem.config.js
pm2 save
pm2 startup
```
**تبریک! سرویس شما اکنون آماده دریافت درخواست از سمت اپلیکیشن‌های شماست.**

## 📡 مستندات جامع API

تمام درخواست‌هایی که به سرور ارسال می‌کنید باید شامل هدر (Header) تایید هویت باشند.

**هدرهای الزامی:**
- `Content-Type: application/json`
- `X-API-KEY: [همان رمزی که در فایل .env گذاشتید]`

### ۱. ارسال پیام (Send Message / OTP)
- **مسیر:** `POST /send-otp`
- **نمونه درخواست (Body):**
```json
{
  "phone": "989123456789",
  "message": "کد تایید شما برای ورود به سایت: 12345",
  "session_id": "line-1" 
}
```
> **نکته:** شماره موبایل را همراه با کد کشور (بدون صفر و +) وارد کنید. مقدار `session_id` اختیاری است.

- **نمونه پاسخ موفق (200 OK):**
```json
{
  "success": true,
  "via": "line-1",
  "sender": "989100000000",
  "status": "Sent"
}
```

### ۲. بررسی وضعیت خطوط (Sessions Status)
- **مسیر:** `GET /status`
- **توضیح:** مشاهده وضعیت اتصال خطوط (اینکه آیا واتس‌اپِ روی سرور وصل است یا قطع شده).

### ۳. بررسی سلامت سرور (Health Check)
- **مسیر:** `GET /health`
- **توضیح:** مناسب برای مانیتورینگ آپ‌تایم و مصرف رم سرور.

### ۴. راه‌اندازی مجدد سشن (Logout / Reset)
- **مسیر:** `DELETE /session/line-1`
- **توضیح:** این دستور اتصال واتس‌اپ را قطع کرده، کش‌ها را پاک می‌کند و آماده دریافت اسکنِ QR کد جدید می‌شود.

## ⚙️ مدیریت وقایع (Webhooks)
شما می‌توانید آدرس API سایت خود (مثلاً پنل لاراول) را در فایل `.env` مقابل `WEBHOOK_URL` قرار دهید. 
با این کار، سرور Node.js هر اتفاقی که بیفتد (وصل شدن خط، قطع شدن خط، دریافت کد QR برای اسکن) را در لحظه به سایت شما گزارش (POST) می‌دهد تا نیازی نباشد دائماً ترمینال سرور را چک کنید.
